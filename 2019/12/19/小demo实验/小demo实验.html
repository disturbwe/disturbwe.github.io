搭建项目框架时对vue-router的使用感想

之前学的时候看的视频，对vuerouter理解不深，现在终于搞懂了嵌套路由的作用。

先来看看官方文档怎么描述的：

<div id="app">
  <router-view></router-view>
</div>
const User = {
  template: `
    <div class="user">
      <h2>User {{ $route.params.id }}</h2>
      <router-view></router-view>
    </div>
  `
}
const router = new VueRouter({
  routes: [
    { path: '/user/:id', component: User,
      children: [
        {
          // 当 /user/:id/profile 匹配成功，
          // UserProfile 会被渲染在 User 的 <router-view> 中
          path: 'profile',
          component: UserProfile
        },
        {
          // 当 /user/:id/posts 匹配成功
          // UserPosts 会被渲染在 User 的 <router-view> 中
          path: 'posts',
          component: UserPosts
        }
      ]
    }
  ]
})

可以看到，子路由的页面是被镶嵌到主路由里面的，所以如果切换子路由主路由所显示的界面是不会重新更新的，而且子路由也会继承主路由的路径。

在本项目中，路由结构采用以下结构，做了一些重定向，

import Layout from '@/views/layout/Layout.vue'
const routes = [
  {
    path: '/404',
    name: '404',
    component: () => import('@/views/404.vue')
  },
  {
    path: '*',
    redirect: '/404'
  },
  {
    path: '',
    component: Layout,
    redirect: '/home',
    children: [
      {
        path: 'home',
        name: 'home',
        component: () => import('@/views/home/index.vue')
      },
    ]
  }
]

vue-property-decorator初试

其实不管是用vue-property-decorator还是用scss都是为了让项目的结构更加清晰，便于维护，代码更加优美。

https://www.npmjs.com/package/vue-property-decorator这是vue-property-decorator的npm官网，学习学习。

首先第一感觉是必须要加上

import { Vue, Component, Prop } from 'vue-property-decorator'

然后传统的export default变成了

@Component
export default class *** extends Vue {
    ...
}

props变化也很大

import { Vue, Component, Prop } from 'vue-property-decorator'

@Component
export default class YourComponent extends Vue {
  @Prop(Number) readonly propA: number | undefined
  @Prop({ default: 'default value' }) readonly propB!: string
  @Prop([String, Boolean]) readonly propC: string | boolean | undefined
}
export default {
  props: {
    propA: {
      type: Number
    },
    propB: {
      default: 'default value'
    },
    propC: {
      type: [String, Boolean]
    }
  }
}

这里需要注意的是在@Prop时应该加上ts的类型修饰符！以防编译器报错

watch也类似

import { Vue, Component, Watch } from 'vue-property-decorator'

@Component
export default class YourComponent extends Vue {
  @Watch('child')
  onChildChanged(val: string, oldVal: string) {}

  @Watch('person', { immediate: true, deep: true })
  onPersonChanged1(val: Person, oldVal: Person) {}

  @Watch('person')
  onPersonChanged2(val: Person, oldVal: Person) {}
}

等同于

export default {
  watch: {
    child: [
      {
        handler: 'onChildChanged',
        immediate: false,
        deep: false
      }
    ],
    person: [
      {
        handler: 'onPersonChanged1',
        immediate: true,
        deep: true
      },
      {
        handler: 'onPersonChanged2',
        immediate: false,
        deep: false
      }
    ]
  },
  methods: {
    onChildChanged(val, oldVal) {},
    onPersonChanged1(val, oldVal) {},
    onPersonChanged2(val, oldVal) {}
  }
}

vue中的data就直接在对象中以属性的方式存在，同样，methods也变成了方法

感觉这也算对项目结构的优化吧，毕竟脚手架中和传统面向对象的思维差距太大。

computed是在vue-class-component里面的

// computed
  get computedMsg () {
    return 'computed ' + this.msg
  }

其实看了ts转换成的es5代码后发现类似的计算属性都是通过Object.definePropety()实现的，这个在这里就不多做赘述。
